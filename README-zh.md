## ✨ 核心能力
StoryMuse 实现了结构化的 AI 辅助创作流程，引导创作者从初始概念到可发布的完整作品：

### 🎯 智能创作流水线
- **创意梳理**: 通过引导式提示将一句话创意扩展为结构化的故事核心设定
- **大纲生成**: 遵循经典叙事原则自动生成三幕四段式的完整故事结构
- **人物设计**: 深度人物档案管理，支持动态关系网络映射和演变追踪
- **章节创作**: 上下文感知的写作辅助，内置多版本管理和回滚能力
- **质量保障**: 人物一致性校验、情节漏洞检测、伏笔追踪和平台合规扫描
- **全局可视**: 全面的进度追踪和故事状态可视化

### ⚡ 设计理念
- **隐私优先**: 100% 数据本地存储，任何创作内容都不会上传到外部服务器
- **低上下文优化**: 智能上下文快照将 token 用量控制在 2000 以内，即使是百万字项目也高效运行
- **开发者友好**: 原生 Git 版本管理、纯文本 Markdown 存储、可扩展插件架构
- **协议标准**: 兼容所有支持模型上下文协议（MCP）的 AI 助手和编辑器
- **生产级别**: 针对长篇写作优化，支持 100 万字以上的项目和数百章节的管理

## 🚀 快速开始

### 前置要求
运行本地 MCP 服务需要 Node.js 18+ 版本。如果你还没有安装，请先从 [nodejs.org](https://nodejs.org/zh-cn/) 下载安装。

---
### 方式一：从 Git 仓库安装
如果你想使用最新的开发版本：
1. 打开 Claude Code 编辑器
2. 输入：/plugin marketplace add https://github.com/cvanly2011/StoryMuse.git
3. marketplace安装完成后再输入：/plugin install StoryMuse@storymuse-dev
4. 输入：/reload-plugins命令重新加载插件列表
5. 新建空白文件夹即可开始创作

### 项目结构
StoryMuse 生成清晰、人类可读、工具无关的项目结构：
```
你的小说/
├── story-seed.md             # 核心概念、世界观设定和叙事前提
├── outline.md                # 完整三幕式结构和章节分解
├── characters.md             # 人物档案、成长弧和关系映射
└── chapters/                 # 章节内容，标准格式可直接发布的 Markdown 文件
    ├── 第01章.md
    ├── 第02章.md
    └── ...
```

没有专有格式，没有供应商锁定——所有内容都是你完全拥有和控制的纯文本。

### 架构详细说明
- **客户端接入层**：完全兼容模型上下文协议（MCP）标准，支持所有符合协议的AI助手、编辑器和第三方工具，没有平台绑定，用户可以自由选择创作环境
- **Skill工作流层**：基于Markdown的人类可读工作流定义，提供开箱即用的专业创作流程引导，无需编程即可扩展自定义工作流，适配不同类型的创作需求
- **MCP服务核心层**：所有逻辑100%本地执行，无需上传任何创作内容到外部服务器：
  - 协议适配层：完全遵循MCP协议标准，实现请求解析、响应格式化、错误统一处理等基础能力
  - 中间件层：提供参数校验、响应转换、日志记录等通用能力，确保接口调用的可靠性和一致性
  - 工具集层：实现51个专业创作工具，覆盖内容管理、创作辅助、质量检查、系统管理四大类功能
  - 核心服务层：封装质量检查引擎、上下文快照生成、文件同步、Git集成等复杂业务逻辑，供工具层调用
- **数据持久层**：混合存储架构，兼顾性能、可读性和版本管理能力：
  - SQLite数据库：WAL模式优化，存储小说元数据、人物、大纲、伏笔等结构化信息，提供高效查询能力
  - 本地文件系统：所有创作内容以标准Markdown格式存储，完全人类可读，无任何专有格式，用户完全拥有自己的创作数据
  - Git版本仓库：原生支持Git版本控制，无需额外配置即可实现多版本管理、分支实验、变更追踪等专业能力
- **可选外部依赖**：AI大模型调用仅用于内容生成建议等增强功能，所有核心创作管理功能完全可以离线使用，严格保障用户隐私安全

### 关键技术特性
- **WAL 优化的 SQLite**: 大型写作项目读写性能提升 10 倍
- **NLP 驱动的上下文管理**: 智能摘要保持上下文窗口小而相关
- **文件监控守护进程**: 本地文件编辑和内部元数据数据库自动同步
- **分支感知存储**: 原生 Git 架构支持并行故事版本实验
- **可扩展规则引擎**: 基于插件的平台规则系统和质量检查器

## 📖 标准工作流
StoryMuse 实现了经过专业作家验证的成熟创作流程：

### 1. 创意阶段
使用 `/story-seed` 模块完善你的核心概念，包括：
- 叙事主题和核心冲突
- 世界观规则和设定边界
- 人物动机和成长弧光
- 独特卖点和受众定位

### 2. 大纲阶段
使用 `/story-skeleton` 生成完整的叙事结构：
- 标准三幕式结构（铺垫→对抗→解决）
- 四级层级（幕→卷→章→场景）
- 章节级目标、冲突点和转折点
- 伏笔埋设和回收计划

### 3. 人物开发阶段
使用 `/story-characters` 构建层次分明、逻辑自洽的人物：
- 包含内部冲突、动机和成长弧光的人物档案
- 动态关系网络，追踪跨章节的关系演变
- 整个叙事过程中的人物一致性自动校验

### 4. 写作阶段
使用 `/story-write` 逐章创作你的小说：
- 自动加载上下文，包括核心设定、最近事件和未解决的情节线索
- 多版本章节管理，一键回滚
- 自动提取关键事件和人物状态，用于连续性追踪

### 5. 审核阶段
使用 `/story-check` 确保质量和一致性：
- 对照已建立的人物档案验证人物行为一致性
- 伏笔回收追踪，识别悬而未决的情节线索
- 情节逻辑和时间线验证
- 目标平台合规性和内容政策扫描

### 6. 发布阶段
所有内容都是标准 Markdown——无需转换或导出，直接发布到任何平台。

## 🎯 平台适配
内置支持所有主流网络小说发布平台，自动规则校验：
- **内容合规**: 特定平台的敏感词扫描和内容政策验证
- **结构优化**: 根据平台受众提供章节长度和叙事节奏建议
- **风格指导**: 根据不同人群期望调整写作风格
- **审核预检查**: 提前扫描常见的审核拒稿原因

目前支持：起点、晋江、番茄小说、七猫、纵横、17K、豆瓣阅读、阅文全平台。

## 🌿 原生 Git 版本控制
StoryMuse 从设计之初就与 Git 版本管理无缝集成：
- **多情节线实验**: 使用独立分支尝试不同的故事方向，不影响主草稿
- **多结局创作**: 在隔离分支中编写和比较多个结局
- **多平台适配**: 维护针对不同发布平台要求优化的独立版本
- **多人协作**: 多位作者可以使用标准 Git 工作流同时创作故事的不同部分

没有专有版本控制系统——使用你已经熟悉和喜爱的工具。

## 🛠️ 开发与扩展
StoryMuse 完全开源且可扩展：

### 从源码构建
```bash
git clone https://github.com/cvanly2011/StoryMuse.git
cd story-muse/mcp-server
npm install
npm run build
```

### 扩展功能
- **添加平台规则**: 在 `mcp-server/src/config/platform-rules/` 目录中添加 JSON 配置文件
- **创建新工作流**: 在 `skills/` 目录中添加 Markdown skill 定义
- **扩展检查引擎**: 在 `mcp-server/src/services/check.service.ts` 中添加新的质量检查逻辑
- **自定义集成**: 扩展 MCP API 以集成到你的写作工具链

查看[开发者指南](docs/developer-guide.md)获取完整文档。
